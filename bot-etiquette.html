When sending automated requests to Weasyl (whether on behalf of a user or not), to help us keep track of and manage malicious and unwanted traffic, we’d appreciate it if you:

  • Use an identifying User-Agent header that doesn’t imitate a browser.

    Yes

    Example-App/1.0 (+https://domain.example/page-with-info-about-your-app)

    No

    Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:128.0) Gecko/20100101 Firefox/128.0

    Even if your request comes from a real browser, e.g. from an extension or in headless mode, please take the time to extend the User-Agent header to distinguish your app or bot if technically possible.

  • Send requests at a reasonable rate.

    At most one request per second, say, if sending a large batch of requests. (Small groups of requests as a result of user action don’t need to be rate-limited.) Retry failed requests with exponential backoff.

For any questions/concerns/requests, see the Support section at the end of any Weasyl page.
